\# 📊 Prolongation Analysis



Аналитический проект по оценке эффективности аккаунт-менеджеров через коэффициенты пролонгации.



\## 📌 Цель проекта



Рассчитать показатели пролонгации клиентов:

\- в \*\*первый месяц\*\*

\- во \*\*второй месяц\*\*



и подготовить отчет для принятия управленческих решений.



\---



\## 📂 Данные



Используются 2 источника:



\### 1. prolongations.csv

\- `id` — ID проекта  

\- `month` — месяц завершения проекта  

\- `AM` — аккаунт-менеджер  



\### 2. financial\_data.csv

\- `id` — ID проекта  

\- месяцы — сумма отгрузки  

\- служебные значения: `в ноль`, `стоп`, `end`



\---



\## ⚙️ Логика расчета



\### 🔹 Коэффициент 1

Отношение:

> суммы проектов, продленных в следующий месяц  

к сумме всех завершенных проектов



\---



\### 🔹 Коэффициент 2

Отношение:

> суммы проектов, продленных через месяц  

к сумме проектов, не продленных сразу



\---



\## 🧩 Основные шаги



1\. Очистка данных

2\. Преобразование в long-формат

3\. Конвертация месяцев в datetime

4\. Определение пролонгаций через событийную модель

5\. Расчет коэффициентов

6\. Формирование отчетных таблиц

7\. Выгрузка в Excel



\---



\## 📊 Результат



Формируется Excel-файл с:

\- отчетом по месяцам

\- отчетом по менеджерам

\- матрицами коэффициентов
